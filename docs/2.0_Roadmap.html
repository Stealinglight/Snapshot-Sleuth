The Hybrid Architecture

The core innovation is splitting the workflow. "ColdSnap" uses Lambda for instant, API-based block reading (no mount required), while the "Forensic Worker" uses Fargate for deep, mounted analysis of heavy volumes.

Phase 3: The ColdSnap Advantage

By implementing Direct Block Access, we bypass the need to provision EBS volumes for triage tasks. This results in an exponential reduction in "Time-to-Evidence".

Legacy Method

Create Volume -> Wait for Init -> Attach to EC2 -> Mount -> Read.

Avg Time: 45-90 seconds (plus initialization warm-up)

ColdSnap Method

Snapshot ID -> EBS API -> Get Block -> Parse Bytes.

Avg Time: < 3 seconds

Phase 2: Deep Forensic Analysis

Moving to Fargate allows for CPU-intensive tasks. We introduce Entropy Analysis to detect packed binaries (malware) and Size Discrepancy Checks to find rootkits lying about file size.
